实验楼 之 Linux

新手入门

Hello World

实验要求：在linux环境中书写并编译执行一个c文件，打印“hello world”

  • 在Linux环境中，打开命令行工具，新建一个名为hello.c的文件

    $ touch hello.c

  • 使用gedit打开

    $ gedit hello.c

  • 编写c文件

    #include<stdio.h>
int main(){
    printf("hello world");
    return 0;
}

  • 编译

    $ gcc -o hello hello.c

    会生成一个名为hello的可执行文件

  • 执行

    $ ./hello

    屏幕上便会打印出hello worlds%字样。

Linux 基础入门

什么是Linux系统

计算机系统包括硬件和软件两部分。

在硬件和应用系统之间需要内核和系统调用来链接。

Linux就是一个包含了内核与系统调用的操作系统。

Linux历史

起源

Linux由Linus Torvalds于1991年10月在赫尔辛基大学写成。它以Unix为原型，从另一个类Unix系统Minix下手，开发出了一个完整的操作系统，起名为Linux。

发展

  • C语言与Unix之父 Ken Thompson和Dennis Ritchie
  • GNU之父 Stallman
  • Minix之父 Tanenbaum
  • Linux之父 Linus Torvalds

XFace快捷键

  • Tab：快速补全
  • Ctrl+c：停止当前命令
  • Ctrl+d：退出终端
  • Ctrl+s：暂停当前程序，之后按任意键继续执行
  • Ctrl+z：将当前程序放到后台执行，恢复为fg
  • Ctrl+a：将光标移至行头
  • Ctrl+e：将光标移至行末
  • Alt+BackSpace：向前删除一个单词
  • Shift+pgUp：将终端向上移动
  • Shift+pgDown：将终端向下移动

通配符

通配符主要有*和？，只用在参数里，不能用在指令里。

举例

  • 新建两个文件

    $ touch abderf.txt sefrt.txt

  • 不记得两个名称了，直接用通配符*查找

    $ ls *.txt

  • 一次性创建多个文件名类似的文件

    $ touch love_{1..10}_linux.txt

各种通配符及其用法

    • 0或多个字符
  • ？ 任意一个字符
  • [list] 匹配list中任意一个单一字符
  • [!list] 匹配除list中任意单一字符以外的字符
  • [c1-c2] 匹配c1-c2中任意单一字符 如[0-9] [a-z]
  • {string1..string2} 匹配string1到string2之间的所有字符串

获取帮助

使用man命令

$ man <command-name>
$ man man

作业

$ banner linux

用户及权限管理

用户管理

Linux是一个可以实现多用户操作的操作系统。两个人可以同时登录同一个操作系统，他们共享一些资源，但是又分别有自己的用户空间。

实际上，他们各自的文件等资源是放在同一个物理磁盘里的，是由Linux的用户管理和权限机制来进行的划分，使得在外观上，不同的用户是互相隔离的。

查看用户
$ who  am i
shiyanlou pts/0 2016-10-29 12:46 (:1 0)
或者

$ who mom likess

命令解释：who命令返回的第一列为打开当前终端的用户的用户名。直接使用指令whoami也可以得到这个。第二列是当前的伪终端名。

who命令其他参数

  • -a 打印能打印的全部
  • -d 打印死掉的进程
  • -m 同am i, mom likes
  • -q 打印当前登录用户数及用户名
  • -u 打印当前登陆用户信息
  • -r 打印运行等级
创建用户

在linux系统中，root拥有至高无上的权力。

root是Linux和Unix系统中超级管理员的用户账户，它可以操作其他所有的用户，所以黑客在入侵系统时，往往把自己提升为root用户。

我们登陆系统时一般使用一般用户，切换root用户需要使用sudo指令。

使用sudu指令有两个前提：一个是你要知道当前登录用户的密码,二是当前用户必须属于sudo用户组。

su su- sudu

  • su 可以切换用户user，执行时需要输入密码
  • su- 也是切换用户，同时环境变量也会跟着改为目标用户的环境变量。
  • sudo 切换为root

实验：

  • 新建用户

    $ sudo adduser lilei
  • 输入密码
  • 确认密码
  • 一路回车

  • 切换用户

    $ su -l lilei
  • 输入密码。命令行提示会改变

  • 退出当前用户

    $ exit
用户组

Linux的每个用户都有自己的用户组。每个用户可以属于多个用户组。新建用户时如果不指定，会再新建一个同名的用户组。

实验：

  • 查看自己所属的用户组

    $ groups shiyanlou

  • 查看/etc/group文件

    $ cat /etc/group/ | sort

etc/group的内容包括用户组（Group）、用户组口令、GID及该用户组所包含的用户

  • 将其他用户加入sudo用户组

文件权限

查看文件权限
$ ls -l

得到的结果每一行从左到右依次为：

文件权限 链接数 所有者 所在用户组 文件大小 最后修改时间 文件名

文件权限

划分：

  • r 读权限：可以通过cat等命令读取
  • w 写权限：可以编辑和修改
  • x 允许执行权限：可以运行的二进制文件或脚本文件
  • d 目录
  • l 软连接
  • b 块设备
  • c 字符设备
  • s socket
  • p 管道
    • 普通文件

文件权限共10位，从左到右依次为：

  • 1位：是否目录
  • 2-4位：所有者权限
  • 5-7位：所属用户权限
  • 8-10位：其他用户权限

Linux目录结构

Linux以树形目录结构的形式来构建整个系统

Linux磁盘是挂载在目录上的。

FHS系统

FHS：Filesystem Hierachy Standard 文件系统层次结构标准

FHS定义了两层规范

  • 第一层定义了/ 下的各个目录应该放置什么类型的文件。比如/etc应该放设置文件,/bin与/sbin应该放可执行文件。
  • 第二层针对/user及/var这两个目录的子目录来定义。比如/var/log放置系统登录文件、/user/share放置共享数据。

文件路径

和windows的意思一样，就是如何找到一个文件的路径。

表示方法：

  • . 当前目录
  • .. 上级目录
    • 上一次所在的目录
  • ~ 当前用户的home目录
  • pwd 输出当前所在路径（绝对路径）

绝对路径与相对路径

  • 绝对路径就是以根目录为起点的路径
  • 相对路径是相对于你当前的目录的路径。
